## 📖 Descripción del Proyecto

Tras 65 años de liderazgo global, el sistema de datos de **MattelUCAB** ha llegado a su límite técnico. El proyecto **"Dream Legacy"** nace como una solución de ingeniería de alto nivel diseñada para estructurar el ecosistema de la corporación, desde la taxonomía detallada de moldes y ediciones limitadas hasta la compleja lógica de compatibilidad de accesorios y la especulación financiera en el mercado de coleccionistas.

Este sistema funciona bajo una dualidad arquitectónica: opera simultáneamente como un **ERP Corporativo (Back-Office)** y como un **Portal de E-Commerce (Front-Office)**, permitiendo gestionar el ciclo de vida completo del producto (PLM) y las interacciones con el mercado secundario.

---

## 🛠️ Especificaciones Técnicas y Alcance

El sistema está diseñado para soportar las siguientes fases críticas de la operación técnica:

* **🧬 El Genoma Barbie:** Catalogación del ADN de cada unidad, diferenciando moldes de rostro, tonos de piel, tipos de cuerpo y patentes históricas.
* **🏭 Manufactura y Ensamblaje:** Gestión de la Lista de Materiales (BOM) y trazabilidad completa desde la materia prima hasta el lote final.
* **📦 Logística y Distribución Global:** Control de inventarios en fábricas y Hubs regionales, manejando jerarquías de empaquetado (SKU Unitario y Caja Máster).
* **🛒 Omnicanalidad B2B y B2C:** Manejo de órdenes de compra masivas para Retail Partners y ventas directas con membresías "Barbie Signature".
* **⚖️ Mercado Secundario y Subastas:** Módulo avanzado de subastas con reglas *Anti-Sniper* y valoración dinámica de activos según su estado de conservación.
* **👥 Gestión de Talento Humano (HRM):** Control de nómina, turnos rotativos y bonificaciones por productividad.

## 📂 Contenido del Repositorio

Este repositorio contiene todos los artefactos técnicos requeridos para la validación del sistema:

* **`docs/`**: Documentación técnica, incluyendo el Modelo Entidad-Relación (notación Patas de Gallo), Modelo Relacional y Diccionario de Datos.
* **`scripts/`**: Archivos SQL para la creación de la base de datos (DDL), inserción de datos maestros (DML) y reglas de integridad.
* **`src/`**: Código fuente de la aplicación Web, incluyendo las interfaces de usuario (CRUD) y la conexión al servidor.
* **`procedures/`**: Implementación de la lógica de negocio a través de **Store Procedures**.
* **`reports/`**: Formatos de reportes estratégicos y documentos transaccionales generados con **JasperReports**.
